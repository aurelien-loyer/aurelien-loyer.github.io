My Radar

C CSFML Graphique
Voir sur GitHub

Aperçu du projet

My Radar est un simulateur de contrôle aérien développé en langage C avec la bibliothèque graphique CSFML. Ce projet simule des trajectoires d'avions avec un système de détection de collisions en temps réel.

Durée : 3 semaines
Technologies : C, CSFML, Make
Année : 2024

Description

My Radar est un simulateur de trafic aérien qui permet de visualiser et gérer des trajectoires d'avions sur une carte du monde. Le programme lit un fichier de script contenant des informations sur les avions et les tours de contrôle, puis simule leur mouvement et leurs interactions.

Les principales fonctionnalités du projet comprennent :

  • Affichage d'une carte du monde en arrière-plan
  • Visualisation des avions et de leurs trajectoires
  • Système de détection de collisions entre avions
  • Tours de contrôle avec zones d'influence
  • Interface utilisateur permettant d'activer/désactiver certaines fonctionnalités
  • Simulation en temps réel avec contrôle de la vitesse

Aspects techniques

Implémentation

Le projet est entièrement développé en C avec la bibliothèque graphique CSFML pour le rendu. La structure du code suit une approche modulaire avec différentes composantes :

  • Gestion des entités (avions, tours)
  • Système de parsing des fichiers de configuration
  • Moteur de simulation physique
  • Algorithme de détection de collisions quadratique
  • Gestion des événements utilisateur
  • Rendu graphique avec CSFML

Algorithmes clés

Un des défis principaux était l'implémentation d'un système efficace de détection de collisions. L'approche naïve consiste à vérifier chaque paire d'avions, ce qui donne une complexité de O(n²). Pour optimiser le processus :

  • Utilisation de zones de contrôle pour éviter les vérifications inutiles
  • Implémentation d'un système de quadtree pour améliorer les performances sur les grandes simulations
  • Calcul vectoriel pour déterminer les trajectoires et points d'intersection

Défis et solutions

Optimisation des performances

Le défi majeur était de maintenir des performances fluides même avec un grand nombre d'avions à l'écran. Pour résoudre ce problème, j'ai implémenté :

  • Une structure de données spatiale pour réduire le nombre de vérifications de collision
  • Un système de rendu qui n'affiche que les éléments visibles à l'écran
  • Des optimisations mémoire pour réduire les allocations dynamiques pendant la simulation

Précision des collisions

La détection précise des collisions entre objets en mouvement était complexe. Ma solution comprend :

  • Calcul des trajectoires futures pour anticiper les collisions
  • Prise en compte des différentes vitesses et tailles des avions
  • Système de priorité pour gérer les situations de collision multiple

Extrait de code

// Fonction de détection de collision entre deux avions
bool check_collision(airplane_t *a, airplane_t *b)
{
    if (a == b || a->has_landed || b->has_landed)
        return false;
        
    // Vérifier si les avions sont dans une zone de contrôle
    if (is_in_control_area(a) && is_in_control_area(b))
        return false;
        
    // Calculer la distance entre les avions
    float dx = a->position.x - b->position.x;
    float dy = a->position.y - b->position.y;
    float distance = sqrt(dx * dx + dy * dy);
    
    // Vérifier si la distance est inférieure à la somme des rayons
    return distance < (a->radius + b->radius);
}

Conclusion et perspectives

Ce projet m'a permis d'approfondir mes connaissances en programmation C et en développement graphique avec CSFML. J'ai particulièrement apprécié les défis techniques liés à l'optimisation des performances et à la gestion des collisions.

Pour les améliorations futures, j'envisage d'ajouter :

  • Une interface utilisateur plus complète avec des statistiques en temps réel
  • Un éditeur visuel pour créer des scénarios de simulation
  • Un système de météo affectant les trajectoires des avions
  • Des algorithmes d'évitement de collision plus avancés
