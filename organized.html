Organized

2024
C
Algorithmique

À propos du projet

Organized est un programme écrit en C qui permet d'organiser et de trier des données selon des règles prédéfinies. Ce projet a été développé dans le cadre de ma formation à Epitech pour mettre en pratique les concepts d'algorithmique et de manipulation de structures de données.

Voir sur GitHub

Démonstration

Description du projet

Le projet Organized a pour objectif de créer un programme capable de lire un ensemble de données, de les trier selon différents critères, et de les organiser de manière structurée. Ce programme peut être particulièrement utile pour manipuler de grands volumes de données et les présenter de façon lisible et exploitable.

Fonctionnalités principales

  • Lecture de données depuis un fichier d'entrée
  • Tri des données selon différents critères configurables
  • Organisation des données en catégories
  • Affichage des résultats de manière structurée
  • Gestion efficace de la mémoire

Technologies utilisées

C Makefile Structures de données Algorithmes de tri Gestion de fichiers

Compétences développées

  • Conception et implémentation d'algorithmes de tri efficaces
  • Manipulation avancée de structures de données en C
  • Gestion optimisée de la mémoire
  • Traitement de fichiers et parsing de données
  • Organisation et structuration de code complexe

Défis et solutions

Performance avec de grands volumes de données

Un des principaux défis était d'assurer la performance du programme lors du traitement de grands volumes de données. J'ai résolu ce problème en implémentant des algorithmes de tri optimisés et en utilisant des structures de données adaptées pour minimiser la complexité temporelle et spatiale.

Gestion de la mémoire

La gestion efficace de la mémoire était cruciale pour éviter les fuites mémoire et optimiser l'utilisation des ressources. J'ai implémenté une allocation dynamique contrôlée et veillé à libérer correctement la mémoire après utilisation.

Structure modulaire

Pour faciliter la maintenance et l'extension du code, j'ai adopté une approche modulaire avec des composants bien définis et des interfaces claires entre les différentes parties du programme.

/*
** Fonction principale de tri qui appelle les algorithmes appropriés
** selon les critères spécifiés
*/
void sort_data(data_t *data, sort_criteria_t criteria) {
    switch (criteria.algorithm) {
        case SORT_QUICK:
            quick_sort(data->items, 0, data->size - 1, criteria.compare_func);
            break;
        case SORT_MERGE:
            merge_sort(data->items, 0, data->size - 1, criteria.compare_func);
            break;
        case SORT_HEAP:
            heap_sort(data->items, data->size, criteria.compare_func);
            break;
        default:
            // Tri par défaut
            quick_sort(data->items, 0, data->size - 1, criteria.compare_func);
    }
}

/*
** Fonction d'organisation des données en catégories
*/
void organize_data(data_t *data, category_t *categories, int nb_categories) {
    for (int i = 0; i < data->size; i++) {
        for (int j = 0; j < nb_categories; j++) {
            if (match_category(data->items[i], categories[j])) {
                add_to_category(&categories[j], data->items[i]);
            }
        }
    }
}
